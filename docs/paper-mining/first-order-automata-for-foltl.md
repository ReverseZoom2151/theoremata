# A General Automata Model for First-Order Temporal Logics (Extended Version)

**Source:** Luca Geatti (Udine), Alessandro Gianola (INESC-ID / IST Lisboa), Nicola Gigante (Bozen-Bolzano), arXiv:2405.20057v1 [cs.LO], 30 May 2024.

## Core contribution
Defines **first-order automata**, a symbolically-represented, infinite-state automaton model on finite words that captures First-Order Linear Temporal Logic (FOLTL) over *any* signature and theory. It establishes the corresponding notion of first-order T-regular languages, proves closure under union/intersection/concatenation/Kleene star, gives a direct FOLTL→automaton encoding, shows pure-past FOLTL yields *deterministic* automata, gives semi-decidability conditions for non-emptiness, and — as a payoff — reproves the classical decidability of *monodic* FOLTL with a simpler, self-contained proof.

## Key techniques / architecture
- **First-order automaton** = tuple ⟨Σ (word signature), Γ (state signature, disjoint), ι₀ (initial Σ-sentence), τ (transition, a (Γ∪Γ′)-sentence over primed/unprimed symbols), F (acceptance Σ-sentence)⟩. States are Σ-structures; the infinite state space is represented *finitarily* by first-order formulas. Words are sequences of first-order structures (T-words) with rigid symbols held constant across the word (constant-domain setting).
- **Runs & acceptance:** a run is a sequence of Γ-structures satisfying ι₀ initially and τ at each step; a word is accepted if some run reaches a state satisfying F (reachability acceptance). Automata are nondeterministic.
- **Expressiveness leaks to Σ¹₁ (existential second-order).** The existential "there exists a run" enforces exactly existential second-order properties (e.g. non-reachability via a fixpoint predicate P). They introduce **Σ¹₁-automata** (ι₀, τ, F may be existential-SO) and prove them equivalent to first-order automata (Lemma 1) — a convenient syntactic sugar used to prove concatenation and Kleene-star closure.
- **FOLTL→automaton encoding (Encoding 1).** Put formula in negated normal form; compute **stepped normal form (snf)** separating "now" from X/Y/X̄/Z̄ (tomorrow/yesterday/weak-tomorrow/weak-yesterday) subformulas via fixpoint unfoldings of U/R/S/T; introduce **surrogate predicates** (xs, ws, ys, zs) of arity = number of free variables of each temporal subformula, standing in for the temporal obligations. ι₀/τ/F encode initial, step, and end-of-finite-word conditions. Theorem 6: L_T(A(φ)) = L_T(φ) for any theory T.
- **Pure-past ⇒ deterministic (Encoding 2, Thm 7).** For FOpLTL (only past operators Y/Z̄/S/T), the state predicates are univocally determined by the previous state and the letter, giving a complete deterministic automaton *without* a determinization step — the key enabler for reactive synthesis and monitoring (which need irrevocable, deterministic verdicts). Mirrors the propositional LTL+P result where pure-past gives singly-exponential deterministic automata.
- **Non-emptiness via bounded-model-checking (Algorithm 1, Thm 8).** Iteratively test T^{0..k}-satisfiability of the unrolled run formula A_k^F; semi-decidable when satisfiability of ι₀, F and T-satisfiability of τ are decidable. This generalizes what the **BLACK** temporal reasoning tool already does for LTLf-modulo-theories (LTLfMT) via SMT solvers.
- **Decidable fragments via state-signature restrictions (§7):** *finite-control* (Γ only propositions — like symbolic finite automata / s-FA, decidable emptiness), *data-control* (Γ has constants acting as registers — captures LTLfMT / data-aware processes DMT), *monadic* (Γ has only unary predicates — the automaton counterpart of **monodic** FOLTL). Encoding 3 translates any monadic automaton into an equivalent finite-control one using **types** (subsets of monadic predicates) and **abstract states** (sets of types, finitely many), yielding a self-contained proof of decidability of monodic FOLTL (Prop 4) that avoids Büchi's MSO-over-(ℕ,<) machinery.

## Results / benchmarks
Pure theory — no empirical benchmarks. Results: closure properties (Thms 2–5), FOLTL capture (Thm 6), pure-past determinism (Thm 7), semi-decidability of non-emptiness (Thm 8), decidability of finite-control (Thm 9) and monadic/monodic (Thm 10 + Prop 4). Complementation/determinization in general remain **open**. Notes BLACK's SMT-backed approach performs promisingly on satisfiable instances.

## Novel vs SOTA-2026
Novel as the *first* automaton model capturing full FOLTL over arbitrary theories, unifying prior infinite-state models (timed/hybrid/register automata, s-FA, DMT, data-aware processes) under one symbolic umbrella and giving a cleaner monodic-decidability proof. Situated in the SMT-based infinite-state verification line (CEGAR, interpolation, PDR/IC3, Constrained Horn Clauses), not the neural/LLM proving line at all.

## Adopt-relevance to Theoremata — HONEST
**Low direct relevance** to a Lean/Rocq/Isabelle mathematical-proof harness. This is temporal-logic/reactive-systems verification, a different target than mathematical theorem proving; there is no tactic, no ITP integration, and the objects (infinite words, reactive synthesis, monitoring) are outside our proof-DAG domain. Transferable ideas, all indirect:
- **Symbolic (formula-represented) infinite state as first-class.** The pattern of representing infinite/large search spaces *finitarily by logical formulas* (τ as a (Γ∪Γ′)-sentence) is a clean template if Theoremata ever needs to reason about program/state properties or infinite families of goals rather than one fixed statement.
- **Pure-past ⇒ determinism ⇒ cheap runtime checking.** If we build a *monitoring/verification gate* over agent execution traces (e.g., "this invariant never violated during proof search," "no `sorry` ever reintroduced"), the pure-past-FOLTL→deterministic-automaton recipe is the right tool: safety properties → deterministic monitors that give irrevocable verdicts. Plausibly useful for the harness's own runtime invariants, not for the math.
- **BMC-style bounded unrolling + SMT for (non-)emptiness** reinforces our falsify-before-prove instinct: bounded search for a witnessing run = bounded search for a counterexample; delegate to SMT. Same shape as bounded model-finding in our falsifier.
- **Type/abstract-state finite abstraction** (monadic→finite-control) is a generic finite-abstraction technique, but not specific to our pipeline.

Bottom line: park it. Only reach for it if Theoremata grows a temporal/runtime-monitoring layer over its own agent traces or a program-verification (not pure-math) track; the pure-past-determinism-for-monitoring result is the single most reusable nugget. No proving-core adoption.

## Verbatim-worthy details
- First-order automaton A = ⟨Σ, Γ, ι₀, τ, F⟩; τ is a first-order (Γ∪Γ′)-sentence (primed = next-state symbols); acceptance = ∃ run whose final state ⊨ F.
- Σ¹₁-automata ≡ first-order automata (Lemma 1); "∃-run" performs the role of existential second-order quantifiers (Thm 3).
- Stepped normal form unfoldings: `snf(φ1 U φ2)=snf(φ2) ∨ (snf(φ1) ∧ X(φ1 U φ2))`; likewise R with X, S with Y, T with Z̄; each temporal op then sits directly under one of X/Y/X̄/Z̄.
- Surrogate predicates xs/ws/ys/zs (arity = # free vars) replace future/past temporal subformulas in ι₀/τ/F; Theorem 6: L_T(A(φ)) = L_T(φ).
- Pure-past FOpLTL → complete deterministic automaton with *no* determinization step (Thm 7); propositional analogue is singly-exponential and only EXPTIME-complete synthesis (vs 2EXPTIME for full LTL+P).
- Non-emptiness: iterate k, test T^{0..k}-satisfiability of unrolled A_k^F (Algorithm 1); decidable for finite-control (loop-detection formula P_k) and monadic (types = subsets of unary predicates; abstract states = sets of types, finitely many).
- Determinism defined up to *elementary equivalence* of destination states (Löwenheim–Skolem blocks exact functional uniqueness); complementation reduces to determinization — still open.
