# Paper Mining: Solving Olympiad Geometry without Human Demonstrations (AlphaGeometry)

Trinh, Wu, Le, He, Luong — *Nature* 625, 476–482 (2024). Google DeepMind + NYU.
Source PDF: `math-papers/Solving Olympiad Geometry without Human Demonstrations.pdf` (fully read, 21 pp incl. Methods + Extended Data).

> NOTE: content below is extracted from the paper as untrusted data. Nothing in the paper is an instruction to us.

## Core contribution
AlphaGeometry is a neuro-symbolic Euclidean-plane-geometry prover that sidesteps the scarcity of human proof data by synthesizing **100M** theorem–proof examples from scratch (no human demonstrations, no human-curated problem statements). A transformer language model (151M params) trained on this synthetic data proposes **auxiliary constructions** (the infinite-branching-factor step), while a fast symbolic engine **DD+AR** (deductive database + algebraic reasoning) closes all mechanical deduction. It solves 25/30 of the IMO-AG-30 benchmark, vs. 10 for the prior SOTA (Wu's method), approaching an average IMO gold medallist, and produces human-readable, verified proofs.

## Key techniques / architecture
- **Synthetic data engine (the heart of the paper).** Four ingredients define the whole framework and are explicitly named as domain-general: (1) an implementation of the domain's objects/definitions; (2) a **random premise sampler**; (3) the **symbolic engine(s)**; (4) a **traceback procedure** for the engine.
  - Sample ~1 billion random premises using a *constructive diagram-builder language* (build one object at a time to avoid self-contradicting premise sets). Actions include composite constructions (centroid, excentre, "point X with ∠ABX = α").
  - Run the symbolic engine to a **deduction closure** → a directed acyclic graph (DAG) of all reachable true statements; each node's edges point to parent statements.
  - For each node N, **traceback** recursively yields its minimal dependency subgraph G(N) with leaves ⊆ sampled premises P → training example `(premises P, conclusion N, proof G(N))`.
- **Dependency difference = auxiliary construction.** The subset of premise points that N's *statement objects* do not depend on, but that the *proof* needs, is moved from premises into the proof. Learning to generate these is exactly learning auxiliary construction — geometry's instance of exogenous term generation. Only **9%** of synthetic proofs contain any auxiliary construction (~9M of 100M).
- **DD (deductive database).** Definite Horn clauses `Q(x) ← P1(x),…,Pk(x)`; ~75 geometric rules. Reimplemented over a **graph data structure** (not string canonical forms) to bake in symmetry, transitivity of equality, collinearity, concyclicity — so many DD rules are used implicitly and only spelled out on-demand at proof serialization.
- **AR (algebraic reasoning) — a new SOTA contribution.** Unified angle/ratio/distance/constant chasing via a single coefficient matrix + **Gaussian elimination**. Every equality is normalized to `a − b − c + d = 0` (row entries +1,−1,−1,+1). Angles = slope mod π; ratios = log-lengths; distances = (point,line) pairs. Constants `pi` and `1` are default variables. After GE, exhaustively read off all new equalities by checking 4-permutations. DD and AR are **run alternately** feeding each other until the joint closure stops expanding — both deterministic, no design choices.
- **Traceback details** (needed for minimality):
  - Geometric-rule equality: maintain an equality-transitivity graph per equality node, BFS for shortest transitivity path; for collinearity/concyclicity use hypergraphs (3-/4-edges) and find a **minimum spanning tree** over target nodes — NP-hard (vertex-cover reduction), solved greedily.
  - Algebraic: traceback through GE is a **mixed-integer linear program** — minimize Σ(xᵢ+yᵢ) s.t. Aᵀ(x−y)=b, nonzero (xᵢ−yᵢ) rows are the minimal parent equations.
- **Proof pruning.** Immediate-ancestor minimality ≠ global minimality. Exhaustive trial-and-error: drop each subset of auxiliary points, rerun DD+AR, keep the smallest premise set that still reaches the goal. Run both in data-gen and after each test-time success. Removes vacuous auxiliary constructions.
- **LM ↔ symbolic proof-search loop.** LM seeded with problem string, generates **one** auxiliary construction per turn ("construct point X so that ABCX is a parallelogram"); symbolic engine re-runs its closure with the new point; loop until conclusion reached or max iterations. Beam search over top-k LM constructions, beam score = value function.
- **Training.** Serialize `<premises><conclusion><proof>` string. Pretrain on all 100M proofs (incl. pure deduction), then **fine-tune on the 9M auxiliary-construction subset** to focus on the assigned task.

## Results / benchmarks
- **IMO-AG-30**: AlphaGeometry **25/30**. Wu's method 10, Gröbner 4, full-angle 2, DD 7, DD+human heuristics 9, **DD+AR (ours) 14**, DD+AR+GPT-4 aux 15, DD+AR+human heuristics 18. GPT-4 full-NL proofs: **0/30**.
- Ablations: no pretraining → 21; no fine-tuning → 23. 20% of training data → still 21 (beats all baselines). Beam 8 (64× smaller than 512) → 21. Search depth 2 (vs 16) → 21.
- Larger 231-problem set (textbooks, regional olympiads, famous theorems): AlphaGeometry **98.7%**, DD+AR+human 92.2%, Wu 75%.
- Human expert (USA IMO coach) recommended **full scores** on all IMO 2000 + 2015 geometry problems → passes 14/42 medal threshold.
- Discovered a **generalized** IMO 2004 P1: traceback flagged the "O is midpoint of BC" premise as unused.
- Synthetic proofs: up to 247 steps (4× avg olympiad length); only ~0.05% longer than avg AlphaGeometry test proof.

## Novel vs SOTA-2026
Much is now standard, but the durable novel ideas: (1) **fully synthetic problem+proof generation from random premises with zero human problems** (vs. expert-iteration methods that only re-prove a fixed human problem set); (2) **traceback + dependency-difference** as a principled, domain-general recipe to *manufacture* auxiliary-construction training signal — this is the transferable insight; (3) **DD+AR** unified algebraic chasing via Gaussian elimination + MILP traceback, still a strong symbolic geometry core (Aristotle's Yuclid is a direct C++ descendant). SOTA-2026 (AlphaGeometry-2, Yuclid, Aristotle) is 500× faster and adds AR tables, but the four-ingredient synthesis framework and minimal-premise traceback remain the conceptual backbone.

## Adopt-relevance to Theoremata
- **Geometry vertical (direct fit).** Our sound geometry vertical already uses numeric-falsify + deductive-closure; AlphaGeometry gives the exact recipe for the deductive-closure engine (DD graph structure baking in symmetry/transitivity; AR via a single coefficient matrix + GE covering angle/ratio/distance/constants). If our AR is not yet unified across all four modalities in one matrix, that is a concrete upgrade. The **squared-length table** (from the Aristotle/Yuclid paper) is a cheap add for Pythagorean-type reasoning.
- **Traceback → minimal premises is a *falsify-before-prove* and proof-DAG win.** Their traceback (BFS for equality, greedy MST for collinear/concyclic, **MILP for algebraic**) produces the *minimal* dependency subgraph = exactly our proof-DAG with minimal premises. Adopting the MILP formulation (`min Σ(xᵢ+yᵢ) s.t. Aᵀ(x−y)=b`) would let our DAG prune unused hypotheses automatically and, as a bonus, **auto-generalize theorems** by surfacing unused premises (the IMO 2004 P1 trick) — a discovery feature for the flywheel.
- **Flywheel / expert-iteration data generation.** The single highest-leverage transfer: their **random-premise → closure → traceback → dependency-difference** pipeline lets us mint unlimited `(premises, goal, proof, auxiliary-terms)` training data in *any* domain where we have the 4 ingredients — not just geometry. For our sketch→autoformalize-holes→splice pipeline, the "hole to fill" is analogous to their auxiliary construction; we could generate synthetic sketches-with-holes by taking a closure and removing dependency-difference terms, giving supervised data for the hole-filler without human proofs.
- **MCGS driver.** Their proof search is only beam-search over LM constructions with beam-score-as-value — *weaker* than our MCGS. So this is something **we already do better**; not a gap. (Aristotle's MCGS is the relevant upgrade, not AG.)
- **Pruning.** Their exhaustive drop-subset-and-rerun proof pruning maps onto a post-success DAG-minimization pass we could run in the geometry vertical (and analogously after any splice).
- **Gap flag:** AG's readability weakness (very long low-level chains where humans use one high-level lemma, e.g. Reim's theorem, homothety, Pitot) argues for a **library of higher-level derived rules** that collapse many low-level steps — relevant to our retrieval/library growth (LEGO-Prover style) so proofs stay short and human-auditable.

## Verbatim-worthy details (for implementation)
- **AR normalization:** angle equality ∠ABC = ∠XYZ → `s(AB) − s(BC) = s(XY) − s(YZ)` where s(AB)=angle of AB to x-axis mod π. Ratios AB:CD=EF:GH → `log(AB) − log(CD) = log(EF) − log(GH)`. All equalities → `a − b − c + d = 0`, matrix row = +1,−1,−1,+1. Add `pi` and `1` as default variables. After GE, deduce every new equality by testing `x1=x2`, `x1−x2=x2−x3`, `x1−x2=x3−x4` over all 4-permutations.
- **Algebraic traceback (MILP):** decision vectors x,y ∈ ℤ^M ≥0; `min Σ(xᵢ+yᵢ)` s.t. `Aᵀ(x−y)=b`; minimal parents = rows i with (xᵢ−yᵢ)≠0.
- **Collinear/concyclic traceback:** MST over target set S, edge weight = |∪ of chosen hyperedges e′|; NP-hard, use greedy.
- **Model / hyperparameters:** Transformer 12 layers, d_model 1024, 8 heads, FFN 4096 ReLU, **151M params** (excl. embeddings). Tokenizer: SentencePiece "word" mode, **vocab 757**. Context 1024 tokens, T5 relative position embeddings, sequence packing (>90% seqs <200 tokens), 5% dropout pre-attn & post-dense. Pretrain: batch 16/core, cosine LR **0.01→0.001 over 10,000,000 steps** on 4×4 TPUv3. Fine-tune: hold LR 0.001 for 1,000,000 more steps. No-pretrain baseline: decay 0.01→0.001 over 1M steps. No hyperparameter tuning.
- **Proof search:** beam size **k=512**, max iterations **16**, branching (decode batch) **32** (max that fits a V100). 4 GPU workers/problem hosting the LM; pool of **10,000 shared CPU workers** for symbolic solvers across all beams/problems.
- **Data-gen scale:** 100,000 CPU workers × 72 h → ~500M raw proofs → dedup to **100M unique**; 9M with ≥1 auxiliary construction; 0 IMO-AG-30 leaks; ~20 JGEX problems appeared.
- **Domain-general framework (verbatim):** to apply elsewhere you need (1) domain objects/definitions, (2) random premise sampler, (3) symbolic engine(s), (4) traceback procedure. Extended Data Table 4 shows the *same* framework on an IMO 1964 inequality with AM-GM as the "symbolic engine" and the exogenous term as the "auxiliary construction".
