## Highest-value items from this slice

Ranked by (soundness first, then how concrete the mechanism is, then how cheap it is):

1. **Route the LeanParanoia per-check failure taxonomy into repair** (PARTIAL). We already capture
   `failures` at `hardening.rs:326-402` and throw the structure away at the gate. The keys
   (`CustomAxioms`, `Replay`, `SourcePatterns`, ...) are exactly the actionable signal a repair step
   wants. This is the last of the five LeanParanoia port gaps still open, and it is nearly free.
2. **Olean byte-corruption integrity check** (NOT-BUILT). A cheap supply-chain smoke test on our
   olean cache, with a proven technique upstream (flip bytes, assert Replay fails). Soundness item,
   no dependencies, small.
3. **Discharge-validity and mutation postconditions on the graph** (PARTIAL). We have taint, cycle
   detection, and scope, but no `assert_valid_graph!`-equivalent running as a postcondition on every
   mutation, and no discharge-balance check. Alethfeld's invariant-at-the-source pattern is what
   stops a slowly-corrupting graph from silently certifying.
4. **Refuted-plan-step adjudication** (NOT-BUILT). When the prover claims a plan step is false, no
   component independently judges that claim. Right now that claim either stalls the loop or is taken
   on faith, and it is the correct discriminator for revise-plan versus rewrite.
5. **Citation and reference verification** (NOT-BUILT, two rows). Two independent sources
   (QED, alethfeld) call citations the number-one hallucination source and both built a checker.
   We have none. This bites the moment we accept informal proofs that lean on literature.
6. **Theorem-Audit as a real pre-proof phase with a refuse verdict** (PARTIAL). We have defect
   priors and statement validation but nothing that can say "this statement smells adversarial,
   refuse". Alethfeld's brokenmath results show this is where detection actually happens, and we
   already ingest brokenmath so the eval loop is ready.
7. **Prover-side forbidden-list and verifier anti-sycophancy prompts** (both PARTIAL). Alethfeld
   derived these from measured benchmark failures, not from taste. We have the graph mechanisms and
   the critic taxonomy but not the generation-time and verification-time prompt content.
8. **Chi-squared distribution test plus decorrelated parallel-worker seeds** (NOT-BUILT /
   UNCERTAIN). Cheap, and the seed question is a concrete possible live bug: if N parallel formalize
   workers share a seed, our best-of-N diversity is fictional and every downstream number that
   assumes independent draws is wrong.
9. **Estimates benchmark track** (NOT-BUILT). ~30 problems, each already a quadruple of informal
   statement, SymPy encoding, expected result, and reference tactic sequence, including negative
   cases with counterexamples. We have the whole Theta/log-linarith/Farkas stack built and no
   benchmark pointed at it.
10. **Reproduction mapping table for eval claims** (NOT-BUILT). One table binding each claim to its
    exact command and output path. The doc says outright "We should adopt this". It costs a
    document and it is what makes every other number here auditable.
